# Project Plan

## Risk Analysis (TAME)

| Risk | TAME Strategy | Plan |
|------|---------------|------|
| Website files are not uploaded correctly to GitHub | **Transfer** | Push changes to GitHub after completing each phase so there is always a backup copy of the project. |
| Some information or images are missing before the project deadline | **Accept** | If a small piece of information cannot be added before the deadline, complete all required project elements first and add optional content only if time allows. |
| Broken links, images, or formatting errors | **Mitigate** | Test every page, image, and navigation link before submitting the project and fix any problems that are found. |
| Running out of time before the deadline | **Eliminate** | Break the project into three phases with a schedule so each part is completed before the final due date. |
